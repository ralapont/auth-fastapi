Control de Errores de la API

Guía de códigos HTTP y estructuras JSON para respuestas de error.

Situación / Error Código HTTP Ejemplo de Respuesta JSON
Credenciales inválidas
Error en login (usuario/pass)
 401 
{
  "error": "invalid_credentials",
  "message": "El usuario o la contraseña no son correctos."
}
Token expirado
Requiere refresh del token
 401 
{
  "error": "token_expired",
  "message": "El token ha expirado. Solicite uno nuevo mediante /refresh."
}
Permisos insuficientes
Acceso no autorizado al recurso
 403 
{
  "error": "forbidden_scope",
  "message": "El usuario no tiene permisos para acceder a este recurso."
}
Usuario no encontrado
ID de usuario inexistente
 404 
{
  "error": "user_not_found",
  "message": "No existe un usuario con el identificador proporcionado."
}

### Manejo de Errores Comunes

| Código | Error | JSON de Respuesta |
| :--- | :--- | :--- |
| `401` | **Credenciales** | `{"error": "invalid_credentials"}` |
| `401` | **Expirado** | `{"error": "token_expired"}` |
| `403` | **Permisos** | `{"error": "forbidden_scope"}` |
| `404` | **No encontrado** | `{"error": "user_not_found"}` |
