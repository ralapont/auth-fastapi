Referencia de Endpoints de la API

Estructura de las peticiones y respuestas para los servicios de la API.

1. JSON de Petición (Request Body)

Acción Endpoint Ejemplo de JSON
Login POST/login 
{
  "username": "juan.perez",
  "password": "MiContraseñaSegura123"
}
Crear usuario POST/users 
{
  "username": "ana.garcia",
  "email": "ana.garcia@example.com",
  "password": "contraseñaSegura!",
  "roles": ["user"]
}
Actualizar usuario PUT/users/{id} 
{
  "email": "ana.garcia@empresa.com",
  "roles": ["user", "admin"]
}

2. JSON de Respuesta (Response Body)

Acción / Endpoint Ejemplo de Respuesta JSON
Login Success
POST/login
 
{
  "access_token": "eyJhbGciOi...",
  "refresh_token": "eyJhbGciOi...",
  "token_type": "Bearer",
  "expires_in": 3600
}
Perfil de usuario
GET/me
 
{
  "id": "a12345",
  "username": "juan.perez",
  "email": "juan.perez@example.com",
  "roles": ["user", "editor"]
}
Detalles de usuario
GET/users/{id}
 
{
  "id": "u789",
  "username": "ana.garcia",
  "email": "ana.garcia@example.com",
  "roles": ["user"],
  "created_at": "2025-11-20T10:15:00Z",
  "active": true
}

Instrucciones de Uso

  • Sustituye {id} por el ID real del usuario.
  • Las respuestas exitosas suelen devolver un código 200 OK o 201 Created.

Notas Técnicas

Los tokens JWT tienen una duración limitada definida en el campo expires_in (en segundos).
