Referencia de Endpoints de la API

Estructura de las peticiones para los servicios de autenticación y gestión de usuarios.

Acción Endpoint Ejemplo de JSON (Body)
Login POST /login 
{
  "username": "juan.perez",
  "password": "MiContraseñaSegura123"
}
Crear usuario POST /users 
{
  "username": "ana.garcia",
  "email": "ana.garcia@example.com",
  "password": "contraseñaSegura!",
  "roles": ["user"]
}
Refresh token POST /refresh 
{
  "refresh_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
Actualizar usuario PUT /users/{id} 
{
  "email": "ana.garcia@empresa.com",
  "roles": ["user", "admin"]
}

Instrucciones de Uso

  • Sustituye {id} por el ID real del usuario en la base de datos.
  • Asegúrate de incluir el header Content-Type: application/json.

Notas Técnicas

El endpoint de Login devuelve un token JWT. Este token debe enviarse en el header Authorization: Bearer [TOKEN] para peticiones protegidas.
